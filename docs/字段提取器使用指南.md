# Java 字段提取器使用指南

## 概述

Java 字段提取器是一个用于从 Java 类中快速提取字段名并生成逗号分隔列表的工具。它可以大大简化 DTO、VO、Entity 等对象之间的字段映射工作。

---

## 功能特点

### ✅ 核心功能
1. **自动识别字段** - 智能识别 Java 字段声明
2. **支持多种修饰符** - private/public/protected
3. **忽略注释** - 自动跳过单行和多行注释
4. **忽略注解** - 自动跳过 @NotNull、@Size 等注解
5. **一键复制** - 支持将结果复制到剪贴板
6. **格式化输出** - 逗号分隔，便于直接使用

---

## 使用方法

### 步骤 1: 打开字段提取器 Tab
- 在应用程序中找到"字段提取"标签页
- 点击进入

### 步骤 2: 粘贴 Java 代码
- 从 IDE 中复制包含字段的 Java 类代码
- 粘贴到输入框中

### 步骤 3: 点击"提取字段"
- 系统会自动分析代码
- 提取所有字段名

### 步骤 4: 查看结果
- 输出框中显示逗号分隔的字段名列表
- 格式：`field1, field2, field3, ...`

### 步骤 5: 复制结果（可选）
- 点击"复制"按钮
- 结果自动复制到剪贴板
- 可直接粘贴到需要的地方

## 示例演示

### 示例 1: 基本字段提取

**输入代码**:
```java
/**
 * id
 */
private Integer todoId;

/**
 * name
 */
private String todoName;
```

**输出结果**:
```
todoId, todoName
```

---

### 示例 2: 带注解的字段

**输入代码**:
```java
/** 用户 ID */
@NotNull
private Long userId;

/** 用户名 */
@Size(max = 50)
private String username;

// 邮箱地址
private String email;
```

**输出结果**:
```
userId, username, email
```

---

### 示例 3: 多种访问修饰符

**输入代码**:
```java
public String name;
private int age;
protected boolean active;
static final String CONSTANT = "value";
```

**输出结果**:
```
name, age, active, CONSTANT
```

---

### 示例 4: 完整类中的字段

**输入代码**:
```java
public class User {
    /** ID */
    private Integer id;
    
    /** 姓名 */
    private String name;
    
    /** 邮箱 */
    private String email;
    
    public void doSomething() {
        // 这是方法，不会提取
    }
}
```

**输出结果**:
```
id, name, email
```

## 支持的字段格式

### ✅ 支持的声明

| 修饰符类型 | 示例 | 是否支持 |
|-----------|------|---------|
| private | `private String name;` | ✅ |
| public | `public int age;` | ✅ |
| protected | `protected boolean active;` | ✅ |
| static | `static final String KEY;` | ✅ |
| final | `final double PI = 3.14;` | ✅ |
| 组合修饰符 | `private static final Logger logger;` | ✅ |

### ❌ 不支持的内容

| 内容类型 | 示例 | 是否提取 |
|---------|------|---------|
| 方法 | `public void doWork() {}` | ❌ |
| 构造器 | `public User() {}` | ❌ |
| 类声明 | `public class User` | ❌ |
| 接口 | `public interface Service` | ❌ |
| 局部变量 | 方法内的变量 | ❌ |

## 实际应用场景

### 场景 1: MyBatis ResultMap 配置

**需求**: 为 Entity 类配置 ResultMap

**原始代码**:
```java
public class User {
    private Integer userId;
    private String username;
    private String email;
    private LocalDateTime createTime;
}
```

**使用字段提取器**:
1. 粘贴代码到输入框
2. 点击"提取字段"
3. 得到：`userId, username, email, createTime`
4. 复制到 MyBatis XML:

```xml
<resultMap id="UserResultMap" type="User">
    <result column="user_id" property="userId"/>
    <result column="username" property="username"/>
    <result column="email" property="email"/>
    <result column="create_time" property="createTime"/>
</resultMap>
```

---

### 场景 2: DTO 转 VO

**需求**: 快速列出需要复制的字段

**原始代码**:
```java
// UserDTO.java
public class UserDTO {
    private String name;
    private Integer age;
    private String phone;
    private String address;
}
```

**使用字段提取器**:
1. 提取字段：`name, age, phone, address`
2. 在转换代码中直接使用:

```java
// 手动或使用 BeanUtils
vo.setName(dto.getName());
vo.setAge(dto.getAge());
vo.setPhone(dto.getPhone());
vo.setAddress(dto.getAddress());
```

## 注意事项

### ⚠️ 重要提示

1. **只提取实例字段** - 不会提取方法中的局部变量
2. **需要分号结尾** - 字段声明必须以分号结束
3. **标准 Java 语法** - 必须符合 Java 语法规则
4. **不处理继承** - 只提取当前类中声明的字段

### 💡 最佳实践

1. **格式化代码** - 粘贴前最好先格式化代码
2. **一个类一次** - 建议每次只处理一个类
3. **检查输出** - 提取后快速检查结果是否正确
4. **配合使用** - 与其他工具配合提高效率

## 常见问题

### Q1: 为什么有些字段没有被提取？
A: 检查字段声明是否完整，必须有分号结尾。确保不是方法或构造器。

### Q2: 支持泛型字段吗？
A: 支持。例如 `private List<String> items;` 会提取为 `items`。

### Q3: 会提取常量吗？
A: 会提取。包括 `static final` 声明的常量。

### Q4: 支持 Kotlin 吗？
A: 当前版本仅支持 Java 语法，不支持 Kotlin。

### Q5: 如何提取父类的字段？
A: 需要分别提取每个类的字段，然后手动合并。

## 技术实现

### 核心算法

1. **行扫描** - 逐行分析代码
2. **模式匹配** - 使用正则表达式识别字段声明
3. **过滤处理** - 跳过注释、注解、方法等
4. **字段提取** - 从字段声明中提取字段名

### 关键代码逻辑

```java
// 匹配字段声明
if (line.matches(".*\\s+(private|public|protected)?\\s*(static)?\\s*(final)?\\s*\\w+\\s+\\w+\\s*;.*")) {
    // 提取字段名
    String fieldName = extractFieldName(line);
}
```
