# JSON 生成器规则文档

## 概述

JSON 生成器是一个用于将特定格式的输入文本转换为 JSON 格式的工具。它支持多种语法规则，包括对象数组、字符串数组、数字类型识别和随机值生成。

---

## 基本语法

### 输入格式

使用逗号分隔字段，支持以下特殊语法：
- `[]` - 方括号表示数组
- `{}` - 花括号表示对象
- 纯数字自动识别为数字类型

---

## 规则详解

### 规则 1：简单对象生成

**语法**: `field1, field2, field3`

**说明**: 简单的逗号分隔字段会生成键值对对象

**示例**:
```
输入: name, age, city
输出:
{
    "name": "name42",
    "age": "age17",
    "city": "city89"
}
```

**注意**: 
- 非数字字段的 value 会随机生成（key + 两位随机数字）
- 每次生成的结果都不同

---

### 规则 2：对象数组

**语法**: `arrayName[{field1, field2}]`

**说明**: 方括号内包含花括号，生成对象数组

**示例 1**:
```
输入: users[{id, name, age}]
输出:
{
    "users": [
        {
            "id": "id23",
            "name": "name67",
            "age": "age91"
        }
    ]
}
```

**示例 2**:
```
输入: items[{sku, price, quantity}]
输出:
{
    "items": [
        {
            "sku": "sku45",
            "price": "price12",
            "quantity": "quantity78"
        }
    ]
}
```

---

### 规则 3：字符串数组

**语法**: `arrayName[element1, element2]`

**说明**: 方括号内直接包含元素，生成字符串数组

**示例 1**:
```
输入: tags[red, green, blue]
输出:
{
    "tags": [
        "red", "green", "blue"
    ]
}
```

**示例 2**:
```
输入: data[x, y, z]
输出:
{
    "data": [
        "x", "y", "z"
    ]
}
```

---

### 规则 4：数字类型

**语法**: 纯数字字段（整数或小数）

**说明**: 
- 纯数字类型的 value 不加引号
- 数字类型不应用随机生成规则
- 支持整数和小数

**示例 1 - 整数**:
```
输入: 100, 200, 300
输出:
{
    "100": 100,
    "200": 200,
    "300": 300
}
```

**示例 2 - 小数**:
```
输入: 1.5, 2.8, 3.14
输出:
{
    "1.5": 1.5,
    "2.8": 2.8,
    "3.14": 3.14
}
```

**示例 3 - 混合类型**:
```
输入: name, age, 100
输出:
{
    "name": "name34",      // 字符串，随机生成
    "age": "age56",        // 字符串，随机生成
    "100": 100             // 数字，不加引号
}
```

---

### 规则 5：随机值生成

**适用范围**: 
- ✅ 简单对象的非数字字段
- ✅ 对象数组中的非数字字段
- ❌ 数字类型字段
- ❌ 字符串数组中的元素

**生成规则**: `value = key + 两位随机数字 (00-99)`

**示例 1 - 简单对象**:
```
输入: product, category, price
可能输出 1:
{
    "product": "product23",
    "category": "category67",
    "price": "price91"
}

可能输出 2:
{
    "product": "product45",
    "category": "category12",
    "price": "price78"
}
```

**示例 2 - 对象数组**:
```
输入: user[{name, email}]
可能输出 1:
{
    "user": [
        {
            "name": "name34",
            "email": "email89"
        }
    ]
}

可能输出 2:
{
    "user": [
        {
            "name": "name67",
            "email": "email23"
        }
    ]
}
```

## 规则对比表

| 输入格式 | 输出类型 | Value 是否随机 | Value 是否加引号 | 示例 |
|---------|---------|--------------|----------------|------|
| `name` | 字符串 | ✅ 是 | ✅ 是 | `"name": "name23"` |
| `100` | 数字 | ❌ 否 | ❌ 否 | `"100": 100` |
| `data[x,y]` | 字符串数组 | ❌ 否 | ✅ 是 | `"data": ["x", "y"]` |
| `obj[{a,b}]` | 对象数组 | ✅ 是 | ✅ 是 | `"a": "a45", "b": "b67"` |
| `items[1,2]` | 数字数组 | ❌ 否 | ❌ 否 | `"items": [1, 2]` |

## 特殊说明

### 1. 随机数的生成
- 随机数范围：00-99
- 每次点击"生成 JSON"按钮都会重新生成
- 同一字段在不同时间生成的结果不同

### 2. 数字识别
- 支持整数：`1`, `100`, `2024`
- 支持小数：`1.5`, `3.14`, `0.99`
- 字母数字组合视为字符串：`a1`, `user123`

### 3. 空格处理
- 输入中的所有空格会被自动移除
- 建议在逗号后添加空格以提高可读性

### 4. 嵌套限制
- 当前版本支持一层对象数组
- 字符串数组不支持嵌套

## 使用方法

1. 在输入框中输入符合上述规则的文本
2. 点击"生成 JSON"按钮
3. 在输出框中查看生成的 JSON 格式
4. 如需重新生成，再次点击"生成 JSON"按钮
5. 使用"清空"按钮清除输入和输出

## 常见问题

### Q1: 为什么我的 value 没有随机生成？
A: 检查字段是否为纯数字。纯数字不会随机生成，会保持原样且不加引号。

### Q2: 如何让数组中的元素也随机生成？
A: 使用对象数组语法 `array[{field1, field2}]`，只有对象数组中的字段会随机生成。

### Q3: 支持多层嵌套吗？
A: 当前版本主要支持一层对象数组。复杂嵌套结构可能无法正确解析。

### Q4: 随机数可以自定义范围吗？
A: 当前固定为两位数字 (00-99)。如需自定义，需要修改源代码。

## 技术实现

### 核心方法

1. **parseToJson(String input)** - 主解析方法
2. **isNumeric(String str)** - 数字识别
3. **generateRandomTwoDigits()** - 随机数生成
4. **parseParts(String input)** - 输入分词

### 关键代码逻辑

```java
// 判断是否为数字
if (isNumeric(field)) {
    // 数字：不加引号，不随机
    json.append("\"").append(field).append("\": ").append(field);
} else {
    // 非数字：随机生成 value
    String randomValue = field + generateRandomTwoDigits();
    json.append("\"").append(field).append("\": \"").append(randomValue).append("\"");
}
```
